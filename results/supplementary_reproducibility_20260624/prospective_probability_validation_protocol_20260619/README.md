# Prospective Tier-1 probability-validation protocol

Created: 2026-06-19T01:45:01+08:00

This folder is a prospective workflow for a future validation campaign. It is not a result folder and it does not add any verified labels, probability weights, or accuracy estimates to the current IJRS manuscript.

The purpose is to define the data that must be collected before the manuscript can make a stronger design-based map-accuracy claim:

1. A documented probability sample with a sampling frame, strata, inclusion probabilities, and design weights.
2. Independent label adjudication, including a third reader for disputed cases.
3. Field or date-stamped VHR anchors for a random subset, with temporal alignment to the 2024 feature year.

Files in this folder:

- `PROSPECTIVE_TIER1_VALIDATION_PROTOCOL_20260619.md`: step-by-step SOP for the future validation campaign.
- `probability_sample_frame_template_20260619.csv`: header-only template for future probability-sample records.
- `third_reader_adjudication_template_20260619.csv`: header-only template for future independent adjudication.
- `field_vhr_anchor_log_template_20260619.csv`: header-only template for future field or VHR anchoring.
- `probability_validation_config_addendum_20260619.yaml`: parameter skeleton with future-protocol status fields only.
- `prospective_protocol_provenance_20260619.json`: provenance for this prospective protocol folder.
